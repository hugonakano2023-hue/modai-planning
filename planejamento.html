Lógica do gating (frustração calculada)

Promoções com 24h atraso: Free vê a promoção que JÁ PERDEU → "se eu tivesse pago, teria pego"

Cupom sem código: Free vê que tem cupom de 15% off mas não consegue copiar → "o desconto tá aqui na minha frente"

Try-on limitado: Free faz 1/mês, fica viciado → quer mais

Resultado: Conversão free → pago por frustração + FOMO, não por feature list.

⚠️ DECISÃO PENDENTE — Fonte de dados dos produtos

De onde vêm os produtos, preços e promoções que aparecem no app? Esta decisão impacta custo, complexidade e velocidade.

OpçãoComo funcionaPrósContrasCusto
A — APIs de Afiliados Lomadee, Awin, Amazon, Shopee, Shein APIs Legal, link comissão embutido, dados estruturados Nem toda loja tem, catálogo limitado R$0
B — Web Scraping Crawlers nos sites das lojas Dados completos, tempo real Zona cinza legal, lojas bloqueiam, manutenção pesada R$100-500/mês
C — Curadoria Manual Equipe seleciona melhores ofertas/dia Rápido, controle total, zero complexidade Não escala, precisa de tempo humano Tempo
D — Híbrido (Recomendado) APIs afiliados + curadoria manual pra completar Melhor de tudo, lança rápido, escala gradual Manual no início R$0 + tempo

Fonte de imagens para swipe (onboarding)

Opção 1: Banco curado de 100-200 fotos de looks categorizados por tags (casual, streetwear, minimalista, elegante, esportivo, vintage).

Opção 2: Usar imagens dos próprios produtos das lojas afiliadas.

Recomendação: Opção 1 pro onboarding (fotos bonitas e diversas). Opção 2 pro feed diário (produtos reais com preço).

Validação de dados

Zod em todas as API routes. Nenhum input do usuário entra sem validação.

Exemplos: email → z.string().email(), budget → z.number().min(0).max(10000), swipe → z.enum(["like","dislike"])

LGPD

Termos de uso + política de privacidade obrigatórios antes do lançamento.

Dados coletados: email, nome, preferências de estilo, foto (try-on). Fotos processadas pela FASHN.ai (verificar política deles).

O que pode dar errado

Riscos + mitigação pra cada um
RiscoImpactoProbabilidadeMitigação
Custo de try-on explode 🔴 AltoMédia Rate limiting rígido por plano. Monitorar diário. Fallback: desabilitar temporariamente.
APIs de lojas mudam/quebram 🔴 AltoAlta Começar com curadoria manual. APIs como complemento, não dependência.
Qualidade do try-on ruim 🟡 MédioBaixa FASHN.ai tem boa qualidade (576x864px). Testar antes com fotos reais.
Algoritmo recomenda mal 🟡 MédioMédia MVP: tags simples + filtragem. Refinamento com mais dados. Botão "não gostei" em tudo.
Ninguém converte free → pago 🟡 MédioMédia Gating agressivo no free. Testar 3 ofertas. Ajustar pricing se necessário.
Problemas jurídicos com scraping 🟡 MédioBaixa Não fazer scraping no MVP. Usar APIs oficiais de afiliados.
FASHN.ai sai do ar / muda preço 🟡 MédioBaixa Backup: Google Vertex AI (créditos startup). Plano B: CatVTON self-hosted.
Asaas webhook falha 🟡 MédioBaixa Retry automático + verificação manual. Logs de todos os eventos.
Foto do try-on vaza/abusa 🔴 AltoBaixa Fotos processadas e descartadas. Não armazenar long-term. LGPD compliance.

Sequência de build

3 entregas — Hugo aprova cada uma antes de avançar

Entrega 1 — Login + Core + Feed

  • Setup projeto + deploy base Vercel
  • Banco de dados (Neon + migrations)
  • Autenticação (Google OAuth)
  • Onboarding completo (swipe + lojas + budget)
  • Feed "Pra Você" com dados curados
  • Bottom navigation (5 abas)
  • Responsivo mobile

Hugo testa: criar conta, fazer swipe, ver feed, navegar

Entrega 2 — Promoções + Try-on + Pagamentos

  • Aba Promoções completa (flash, cupons, stack)
  • Try-on com FASHN.ai
  • Aba Salvos (wishlist + alertas)
  • Perfil completo (tamanhos, estilo, economia)
  • Integração Asaas (planos, PIX, cartão)
  • Feature gating funcionando
  • Trial 7 dias

Hugo testa: promoções, try-on, assinar, pagar, cancelar

Entrega 3 — Landing + Visual + Lançamento

  • Landing page de conversão
  • Visual polido (referências do Hugo)
  • Meta Pixel + analytics
  • Admin básico (adicionar produtos/cupons)
  • Termos de uso + privacidade
  • Performance (Lighthouse > 90)
  • Segurança final (audit OWASP)

Hugo testa: landing, visual, admin. Vitor testa: criativos, anúncios.
